# 재고 조정 용어 및 기능 설계

## 용어 검토

### 사용자 제안 용어
- **생산입고**: 재고를 추가하는 개념
- **생산소모**: 재고를 차감하는 개념 (이카운트 ERP 용어)

### 문제점
1. **"생산입고"**는 실제로는 생산이 아니라 재고 추가/조정 개념
2. **"생산소모"**는 이카운트 ERP 용어이지만, 현재 시스템에서는 판매 이력과 연계된 재고 조정이 더 적합

### 권장 용어

#### 옵션 1: 재고 조정 (Inventory Adjustment) - **권장**
- **재고 추가**: 재고를 증가시키는 조정
- **재고 차감**: 재고를 감소시키는 조정
- **장점**: 
  - ERP 표준 용어
  - 명확하고 직관적
  - 추가/차감 모두 포함

#### 옵션 2: 재고 보정 (Inventory Correction)
- **재고 보정**: 재고 수량을 수정하는 작업
- **장점**: 
  - 수정/보정의 의미가 명확
- **단점**: 
  - 추가/차감 구분이 모호

#### 옵션 3: 입고/출고 조정
- **입고 조정**: 재고를 추가하는 조정
- **출고 조정**: 재고를 차감하는 조정
- **장점**: 
  - 입고/출고 개념과 일치
- **단점**: 
  - 실제 입고/출고가 아닌 조정과 혼동 가능

#### 옵션 4: 판매 이력 기반 조정
- **판매 추가**: 판매 이력을 추가하여 재고 차감
- **판매 삭제**: 판매 이력을 삭제하여 재고 복구
- **판매 수정**: 판매 이력을 수정하여 재고 조정
- **장점**: 
  - 판매 이력과 직접 연계
  - 이력 추적 용이
- **단점**: 
  - 실제 판매가 아닌 조정도 판매 이력으로 표시됨

## 최종 권장안

### 용어: **재고 조정** (Inventory Adjustment)

#### 세부 용어
1. **재고 추가 조정**: 재고 수량을 증가시키는 조정
   - 예: 실물 재고가 시스템보다 많을 때
   - 예: 재고 실사 후 차이 보정

2. **재고 차감 조정**: 재고 수량을 감소시키는 조정
   - 예: 실물 재고가 시스템보다 적을 때
   - 예: 폐기/손실 처리
   - 예: 기초재고 정리 (생산소모 개념)

3. **판매 이력 조정**: 판매 이력을 추가/수정/삭제하여 재고 조정
   - 판매 추가: 재고 차감
   - 판매 삭제: 재고 복구
   - 판매 수정: 재고 재계산

### UI 표시 용어

#### 입고 확정 재고 상세보기
- **판매 이력** 섹션
  - 추가 버튼: "판매 추가" 또는 "재고 차감"
  - 수정 버튼: "판매 수정" 또는 "재고 조정"
  - 삭제 버튼: "판매 삭제" 또는 "재고 복구"

#### 재고 조정 전용 기능 (향후 추가 가능)
- **재고 조정** 섹션
  - 재고 추가 조정
  - 재고 차감 조정

## 기능 설계

### 현재 요구사항
입고 확정 재고 상세보기에서 판매 이력에 추가/수정/삭제하면 재고가 자동으로 조정됨

### 구현 방안

#### 방법 1: 판매 이력 기반 조정 (권장)
- 판매 이력을 추가/수정/삭제하면 자동으로 재고 조정
- 기존 `SalesItem` 엔티티 활용
- 재고 상태 자동 업데이트 (`updateContainerInventoryStatus`)

**장점:**
- 기존 로직 재사용
- 판매 이력 추적 가능
- 재고 계산 정확

**단점:**
- 실제 판매가 아닌 조정도 판매 이력으로 표시됨
- 해결: `SalesItem`에 `adjustmentType` 필드 추가 (NORMAL, ADJUSTMENT)

#### 방법 2: 별도 재고 조정 엔티티
- `InventoryAdjustment` 엔티티 생성
- 판매 이력과 분리된 조정 이력 관리

**장점:**
- 판매 이력과 조정 이력 분리
- 조정 내역 명확

**단점:**
- 새로운 엔티티 및 로직 필요
- 복잡도 증가

### 권장 구현: 방법 1 (판매 이력 기반)

#### 데이터 구조
```typescript
// SalesItem에 조정 타입 추가 (선택사항)
interface SalesItem {
  // ... 기존 필드
  adjustmentType?: 'NORMAL' | 'ADJUSTMENT' | null; // 조정 여부 표시
}
```

#### 기능
1. **판매 추가**
   - 판매 이력 추가 → `SalesItem` 생성
   - 재고 자동 차감
   - 재고 상태 자동 업데이트

2. **판매 수정**
   - 판매 이력 수정 → `SalesItem` 수정
   - 재고 재계산
   - 재고 상태 자동 업데이트

3. **판매 삭제**
   - 판매 이력 삭제 → `SalesItem` 삭제 (또는 상태 변경)
   - 재고 자동 복구
   - 재고 상태 자동 업데이트
