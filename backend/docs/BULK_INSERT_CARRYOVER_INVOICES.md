# 이월잔액 거래명세서 일괄 삽입 가이드

## 개요

엑셀 데이터를 사용하여 이월잔액 거래명세서를 일괄로 생성하는 스크립트입니다.

## 데이터 형식

스크립트는 다음 형식의 데이터를 처리합니다:

- **전화번호** (선택): 고객 전화번호 (예: `010-1234-5678`, `031-123-4567`)
- **업체명** (필수): 고객 업체명
- **발행일** (필수): MM/DD 형식 (예: `01/27`)
- **채권 잔액** (필수): 숫자 (예: `13106720`)

## 고객 매칭 로직

1. **전화번호로 먼저 찾기**: 전화번호가 있으면 전화번호로 고객을 찾습니다.
2. **업체명으로 찾기**: 전화번호로 찾지 못했거나 전화번호가 없으면 업체명으로 찾습니다.
3. **여러 개 매칭 시 스킵**: 
   - 전화번호로 찾았는데 업체명이 다른 경우
   - 업체명으로 찾았는데 여러 개가 나오는 경우
   - 이런 경우는 스킵하고 사용자가 직접 처리해야 합니다.

## 실행 방법

### 1. 데이터 준비

`backend/scripts/bulk-insert-carryover-invoices.ts` 파일의 `data` 배열에 데이터를 입력합니다:

```typescript
const data: InvoiceData[] = [
  { phone: '031-675-8730', companyName: '농업회사법인 주식회사 태백사료', issuedDate: '01/27', balance: 13106720 },
  { phone: '010-4865-0205', companyName: '우아람팜', issuedDate: '01/27', balance: 26916120 },
  // ... 더 많은 데이터
];
```

### 2. 스크립트 실행

```bash
cd backend
npm run script:bulk-insert-carryover
```

### 3. 결과 확인

스크립트 실행 후 다음 정보가 출력됩니다:

- 성공한 건수
- 실패한 건수
- 실패한 항목의 상세 정보 (고객을 찾을 수 없음, 여러 개 매칭됨 등)

## 생성되는 거래명세서 구조

각 거래명세서는 다음과 같이 생성됩니다:

- **거래명세서 번호**: `YYYY/MM/DD-순번` 형식으로 자동 생성
- **발행일**: 입력한 발행일 사용
- **고객**: 전화번호 또는 업체명으로 찾은 고객
- **항목**: 
  - 품목명: "이월잔액"
  - 공급가액: 입력한 채권 잔액
  - 부가세: 0원 (부가세 미적용)
- **전일잔액**: 발행 시점의 거래처 채권 잔액 (자동 계산)
- **비고**: "이월잔액"

## 채권 자동 업데이트

거래명세서 생성 시 거래처 채권도 자동으로 업데이트됩니다:

- 채권이 없으면 새로 생성
- 채권이 있으면 총 판매액과 잔액 업데이트
- 상태 자동 업데이트 (OUTSTANDING, PARTIAL, COMPLETED)

## 주의사항

1. **고객 매칭**: 
   - 전화번호나 업체명이 정확히 일치해야 합니다.
   - 여러 개가 매칭되면 스킵되므로, 먼저 데이터를 확인하세요.

2. **발행일**: 
   - 현재 스크립트는 2026년으로 설정되어 있습니다.
   - 다른 연도를 사용하려면 `parseDate` 함수를 수정하세요.

3. **사용자 ID**: 
   - 기본값은 1입니다.
   - 다른 사용자로 발행하려면 `createCarryoverInvoice` 함수의 `userId` 파라미터를 수정하세요.

4. **트랜잭션**: 
   - 각 거래명세서는 별도의 트랜잭션으로 처리됩니다.
   - 하나가 실패해도 다른 것들은 계속 처리됩니다.

## 실패 처리

스크립트 실행 후 실패한 항목은 다음과 같이 처리하세요:

1. **고객을 찾을 수 없음**: 
   - 고객 정보를 먼저 등록하거나
   - 전화번호/업체명이 정확한지 확인하세요.

2. **여러 개의 고객이 매칭됨**: 
   - 데이터베이스에서 해당 고객을 확인하고
   - 수동으로 거래명세서를 생성하세요.

## 예시

```typescript
// 성공 케이스
{ phone: '031-675-8730', companyName: '농업회사법인 주식회사 태백사료', issuedDate: '01/27', balance: 13106720 }
→ 전화번호로 고객 찾기 성공 → 거래명세서 생성

// 실패 케이스 1: 고객을 찾을 수 없음
{ phone: '010-0000-0000', companyName: '존재하지 않는 업체', issuedDate: '01/27', balance: 1000000 }
→ 고객을 찾을 수 없음 → 스킵

// 실패 케이스 2: 여러 개 매칭
{ phone: '010-1234-5678', companyName: '중복된 업체명', issuedDate: '01/27', balance: 1000000 }
→ 업체명으로 여러 개 찾음 → 스킵
```
