# Cloud SQL 업그레이드 비교: db-f1-micro → db-g1-small

## 📊 사양 비교

### db-f1-micro (이전)
- **CPU**: 공유 vCPU (버스팅, 최대 0.2 vCPU)
- **메모리**: 0.6GB RAM
- **디스크**: 10GB SSD (현재 설정)
- **최대 연결 수**: 약 25개
- **네트워크**: 표준

### db-g1-small (현재)
- **CPU**: 전용 1 vCPU (항상 사용 가능)
- **메모리**: 1.7GB RAM (약 2.8배 증가)
- **디스크**: 10GB SSD (동일)
- **최대 연결 수**: 약 100개 (4배 증가)
- **네트워크**: 표준

## 💰 가격 비교

### 월별 비용 (한국 리전 기준, asia-northeast3)

#### db-f1-micro (이전)
- **인스턴스**: 약 $7.31/월 (₩9,500/월)
  - CPU 시간: $0.007/시간 × 730시간 = $5.11/월
  - 메모리: $0.003/GB/시간 × 0.6GB × 730시간 = $1.31/월
- **디스크 (10GB SSD)**: 약 $1.70/월 (₩2,200/월)
- **총 비용**: 약 **$9/월** (₩11,700/월)

#### db-g1-small (현재)
- **인스턴스**: 약 $25.55/월 (₩33,200/월)
  - CPU 시간: $0.0175/시간 × 730시간 = $12.78/월
  - 메모리: $0.0175/GB/시간 × 1.7GB × 730시간 = $21.72/월
- **디스크 (10GB SSD)**: 약 $1.70/월 (₩2,200/월) (동일)
- **총 비용**: 약 **$27/월** (₩35,400/월)

#### 비용 증가
- **월간 증가**: +$18/월 (+₩23,700/월)
- **연간 증가**: +$216/년 (+₩284,400/년)

*참고: 실제 비용은 사용량, 디스크 크기, 백업, 네트워크 사용량 등에 따라 달라질 수 있습니다.*

## ⚡ 성능 변화

### 1. CPU 성능
- **이전 (f1-micro)**:
  - 공유 vCPU (버스팅)
  - CPU 사용률이 높을 때 제한 (버스팅 크레딧 소진 시)
  - 응답 시간이 느려질 수 있음
- **현재 (g1-small)**:
  - 전용 1 vCPU
  - 일관된 성능 보장
  - CPU 제한 없이 항상 사용 가능
  - **쿼리 응답 시간 30-50% 개선 예상**

### 2. 메모리
- **이전**: 0.6GB
- **현재**: 1.7GB (2.8배 증가)
- **효과**:
  - 더 많은 데이터를 메모리에 캐싱 가능
  - 디스크 I/O 감소
  - 복잡한 쿼리 성능 향상

### 3. 연결 수
- **이전**: 최대 25개
- **현재**: 최대 100개 (4배 증가)
- **효과**:
  - 동시 연결 제한 해소
  - "connection pool exhausted" 에러 방지
  - 더 많은 사용자/인스턴스 동시 처리 가능

### 4. 처리량
- **동시 쿼리 처리 능력**: 2-3배 향상 예상
- **트랜잭션 처리 속도**: 30-50% 향상 예상

## 📈 실제 성능 개선 예상치

### 응답 시간
- **단순 쿼리 (SELECT)**: 10-20ms → 5-10ms (50% 개선)
- **복잡한 쿼리 (JOIN, 집계)**: 100-200ms → 50-100ms (50% 개선)
- **트랜잭션 (INSERT/UPDATE)**: 50-100ms → 30-60ms (40% 개선)

### 처리량
- **초당 쿼리 수**: 약 2배 증가 예상
- **동시 사용자 수**: 25명 → 100명 지원 가능

## 🎯 ROI (투자 대비 효과)

### 비용 증가
- **월간**: +$18/월

### 예상 효과
1. **에러 해결**: 연결 풀 고갈 에러 완전 해소
2. **성능 향상**: 사용자 체감 속도 30-50% 개선
3. **안정성**: 서비스 중단 위험 감소
4. **확장성**: 더 많은 사용자/트래픽 처리 가능

### 결론
- **소규모/중소규모 프로덕션 환경**에 적합
- 연결 풀 문제 해결과 성능 향상을 고려하면 **비용 대비 효과가 충분**
- 사용자 경험 개선으로 인한 **간접적 이익** 고려 필요

## 📝 추가 최적화 권장사항

### 1. 연결 풀 설정 (완료)
- `max: 20` (각 Cloud Run 인스턴스당)
- 총 최대 연결: 10 인스턴스 × 20 = 200개 (Cloud SQL 최대 100개 이하로 안전)

### 2. Cloud Run 최소 인스턴스 설정 (선택사항)
```bash
gcloud run services update ccbio-erp \
  --min-instances=1 \
  --project=balmy-ground-470504-p0
```
- **비용**: +$5-10/월
- **효과**: 콜드 스타트 제거, 첫 요청 응답 시간 개선

### 3. 디스크 크기 모니터링
- 현재 10GB 사용 중
- 필요시 증가 가능 (추가 비용 발생)

## 🔍 모니터링 포인트

업그레이드 후 확인할 지표:
1. **연결 풀 에러**: 더 이상 발생하지 않아야 함
2. **평균 응답 시간**: 30-50% 개선 확인
3. **CPU 사용률**: 전용 CPU로 안정적인 사용률 유지
4. **메모리 사용률**: 더 많은 여유 확인
5. **동시 연결 수**: 최대 20-30개 수준 유지 (여유 확보)

## 📞 비용 관리 팁

3. **디스크 크기 최적화**
4. **모니터링으로 사용 패턴 분석**
